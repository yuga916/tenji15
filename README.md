# TENJI15 — 展示後に、答えは変わる。

展示航走後の直前情報に特化したボートレース分析メディア。
「事前予想」ではなく「直前情報が事前予想と市場(オッズ)をどう裏切ったか=歪み」だけを扱う。
GitHub Pagesで公開できる静的サイト構成。

## コンセプト

- 事前予想は公式にもポセイドンにもある。TENJI15は**展示後15分間に生まれる情報**しか扱わない
- レースページは同一URLで3段階に進化する:
  1. **事前(pre)**: 番組表ベースの暫定評価
  2. **シグナル(signal)**: 展示・進入・オッズ反映の最終結論+変化の差分
  3. **答え合わせ(verified)**: 直前サインが結果にどう効いたかの検証記事(=ストックコンテンツ)
- 検証済みレースの蓄積が「シグナル成績」「前づけアーカイブ」「会場別展示信頼度」のストックページに育つ

## 直前シグナルの種類

| シグナル | 内容 |
|---|---|
| 乖離(gap) | 展示反映後のAI勝率 − 直前オッズ逆算の市場勝率。市場が織り込んでいない歪みを検知 |
| 前づけ(formation) | スタート展示の進入変化を検知し全艇の勝率を再計算 |
| 展示偏差(extime) | 展示タイムを当日分布×節間自己比で偏差値化 |
| スリット(slit) | 展示STから本番隊形を予測し展開確率(逃げ/まくり/差し)を算出 |
| バイアス(bias) | 当日の消化レースから水面傾向をライブ蓄積 |

## 構成

```
site/index.html                     トップ(次の締切・歪みレーダー・答え合わせアーカイブ)
templates/race-detail.template.html レース詳細(3状態を同一URLで描画)
site/assets/styles.css              Night Waterテーマ・艇色バッジ(1白2黒3赤4青5黄6緑)
src/
  types.ts      スキーマ(Race/Entry/Signal/RaceResult、status: pre|signal|verified)
  dataSource.ts mock / official 切替(officialは公式配布ファイル+直前情報の統合箱)
  store.ts      保存層
  pipeline.ts   データ取得を1回実行
  build.ts      静的サイト生成(状態別レンダリング・labs/ストックページ)
.github/workflows/deploy.yml        開催時間帯(JST8-23時)は15分ごとに自動更新
```

## ローカルでの使い方

```bash
# Node.js 22.6以上(依存パッケージ不要)
npm run site      # データ取得(mock) → dist/ 生成
npm run preview   # http://localhost:4200
# dist/index.html を直接ブラウザで開いてもOK(相対パス生成)
```

モックデータは4レース(シグナル点灯×2・展示待ち×1・答え合わせ済み×1)で、
ライフサイクル全状態のデザインを確認できる。

## 実データ稼働(OfficialDataSource・実装済み)

公式配布の番組表(Bファイル)から全24場の実データを生成する。

```
src/officialFetcher.ts  ダウンロード(1日1リクエスト)+LZH解凍(lhasa/lha/7z)+Shift_JISデコード
src/officialParser.ts   寛容パース(固定バイト位置に依存しない正規表現ベース)
src/venues.ts           全24場マスタ(場コード→名称・イン逃げ基準値)
src/model.ts            ベースライン評価(コース×級別×勝率×モーターでpreProb/イン逃げ確率)
fixtures/b-sample.txt   パーサのテストフィクスチャ
npm run test:parser     23項目のスモークテスト(CIでも毎回実行)
```

**有効化手順**: リポジトリの Settings > Secrets and variables > Actions > **Variables** に
`DATA_SOURCE_MODE` = `official` を追加 → 次のビルドから実データで生成。

**フォールバック設計**: 取得失敗時(未配布時刻・障害)は 前回データ維持 → mock の順で退避し、サイトは壊れない。

**初回実データ投入時の注意**: パーサは実ファイル形式との微差があり得る(フィクスチャは公知の形式に基づく)。
Actionsのログで `[official] N場 Mレースをパースしました` と警告件数を確認し、
形式差異があれば `DEBUG_PARSER=1` で未マッチ行を出して `officialParser.ts` の正規表現を調整する。

**次フェーズ(未実装)**: 展示・直前オッズの反映(シグナル点灯)、結果Kファイルの取り込み(答え合わせ自動生成)。

規約状況(2026/7/4確認済み): 配布ファイルに利用条件の明示なし・robots.txt全許可。取得は1日1リクエストの低負荷設計。公式への直リンクは張らない(「営利目的リンクお断り」文言に留意)。公開運用前にBOATRACE振興会への確認送付を推奨。

## ポジショニング(詳細は gambling-media-domain-comparison.md / boatrace-positioning.md)

- 市場の空席「結論×直前」を取る。公式(中立)・日和(ツール)・BOATERS(有料)・ポセイドン(事前)が構造的に埋めていない

- 収益はP1広告→P2プレミアム→P3提携の順
